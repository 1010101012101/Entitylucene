I use OS X Mavericks 10.9.2.  <answer128867> The second launchd job ticket format is almost correct. Only a set of quotes need removing.  h2Quotes/h2  The quotes are no longer needed in the second launchd job ticket, so change:  precode&lt;string&gt;"set volume output volume 0"&lt;/string&gt; /code/pre  …to…  precode&lt;string&gt;set volume output volume 0&lt;/string&gt; /code/pre  h3Grouping Arguments/h3  In this case, the quotes grouped the set of arguments for codeosascript/code to see as one item. Thus codeosascript/code gets two arguments, the flag code-e/code and the script. This meets the requirements from the osascript manual.  When called through strongTerminal.app/strong and thus through the shell, the quotes are needed to perform the grouping. When codeosascript/code gets the arguments, the shell has removed the quotes.  Without quotes, spaces subdivide each item passed via the shell.  When called through a launchd job ticket, the grouping is intrinsic within the array passed as codeProgramArguments/code. In your first ticket, the array contained nine items and in the second ticket, the array contained three.  h2LaunchAgent/h2  Be sure to add this launchd job as a code~/Library/LaunchAgents/code as codeosascript/code requires an active user with a graphical session.  This approach approximates what I do with Power Manager in How to Set the Volume When Logging In.  <answer129207> The second plist should work if you remove the extra quotes, but it only runs the command at login and not after waking up from sleep.  You can use sleepwatcher to run a command after waking up from sleep. Run codesudo -v/code and paste lines like this in a shell:  precodecurl http://www.bernhard-baehr.de/sleepwatcher_2.2.tgz|tar -x sudo mkdir -p /usr/local/sbin /usr/local/share/man/man8 sudo cp sleepwatcher_2.2/sleepwatcher /usr/local/sbin sudo cp sleepwatcher_2.2/sleepwatcher.8 /usr/local/share/man/man8 cp sleepwatcher_2.2/config/de.bernhard-baehr.sleepwatcher-20compatibility-localuser.plist ~/Library/LaunchAgents launchctl load ~/Library/LaunchAgents/de.bernhard-baehr.sleepwatcher-20compatibility-localuser.plist echo $'#!/bin/sh\nosascript -e "set volume output volume 0"'&gt;~/.sleep chmod +x ~/.sleep /code/pre  <comment150907> Thanks.
