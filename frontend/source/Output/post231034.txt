from codedefaults read/code is sent to emstandard output/em in emhuman readable format/em.  The output from codedefaults export/code is put in the file that you specify in the command in embinary format/em.  The problem with your codedefaults export/code example:   ul liyou redirected the emstandard output/em to the file: code~/Desktop/com.apple.sidebarlists.plist/code (using code&gt;/code) and codedefaults export/code doesn't send anything to standard output; the result was a file that had nothing in it./li licodedefaults export/code requires that you specify an output file in the command. Including the redirection operator (code&gt;/code) before the filename redirected emstandard output/em to that file and effectively removed the filename from the command (since redirection of input or output effectively happens prior to command parsing) Thus no output file was specified which resulted in the error message saying that you need "a path to write to"./li /ul  Depending on what changes you are making:  ul liif possible, use the Finder's codePreferences…/code to change things./li liif codePreferences…/code can't accomplish what you need then it's probably easiest to use codedefaults read/code to produce output in a form that you can easily edit and then use codedefaults write 'plist'/code (see strongman/strong documentation) to "install" your changes./li lithere are other ways to do what you want of varying degrees of difficulty./li /ul  I'll just mention that the documentation for the codedefaults/code command states:     Defaults can be structured in very complex ways, making it difficult   for the user to enter them with this command.   <comment282084> What's wrong with `defaults export com.apple.sidebarlists ~/Desktop/com.apple.sidebarlists.plist` (after removing the zero sized file) except that it is the binary form of the file created by `defaults read com.apple.sidebarlists  ~/Desktop/com.apple.sidebarlists.plist`? <comment282090> could I export com.apple.sidebarlists.plist with defaults export command? why did I get zero KB file with defaults export command?
