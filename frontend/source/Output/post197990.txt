It seems that codedefaults/code isn’t comprehensive enough (anymore?) for you to achieve what you want, however you can use PlistBuddy, which on 10.11 is currently found under code/usr/libexec/PlistBuddy/code, to add/remove/edit plist files.    Here are the two commands you’ll can use to add the code_CFURLString/code and code_CFURLStringType/code:  precode/usr/libexec/PlistBuddy -c 'Add "NSToolbar Configuration Browser":"TB Item Plists":8:_CFURLString string "file:///Applications/Chess.app"' ~/Library/Preferences/com.apple.finder.plist  /usr/libexec/PlistBuddy -c 'Add "NSToolbar Configuration Browser":"TB Item Plists":8:_CFURLStringType integer 15' ~/Library/Preferences/com.apple.finder.plist /code/pre  The “8” found in both of the commands above is the array index of the item.    No code_CFURLAliasData/code is generated (as suggested in several places).  I’ve tried creating an empty code_CFURLAliasData/code, but Finder didn’t populate it.  Another issue is that if I add/remove another item using kbdcmd/kbd-drag all the data I’ve added with PlistBuddy gets erased…  Hopefully this information will get someone else partway there.  <comment237085> I recommend using the excellent python script [dockutil](https://github.com/kcrawford/dockutil) for programmatically handling dock items. dockutil --add /Applications/TextEdit.app
