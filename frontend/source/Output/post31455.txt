Change title given to Reading List article <body> I often use Safari's Reading List functionality to save links to pdf documents from the arXiv.  The URL for the particular paper -- and hence the title given in the Reading List -- is not very informative:  img src="https://i.stack.imgur.com/4gJiu.png" alt="example reading list"  Is there a way to change the title of the item in the Reading List?  Perhaps there is some file that stores metadata for this item?    <answer32531> As far as I know, there’s no way to do it directly through the GUI, but there is a file you can edit quite easily. The file code~/Library/Safari/Bookmarks.plist/code contains all of the Safari bookmarks, including those saved to the Reading List. (For whatever reason, this file gets borked in TextMate, among others, but TextWrangler seems to cope just fine).  Anything beginning code&lt;key&gt;ReadingList&lt;/key&gt;/code is an item saved to the Reading List. There are two entries that control the title and the description. The first will be of the form:  precode&lt;key&gt;PreviewText&lt;/key&gt; &lt;string&gt;No preview available&lt;/key&gt; /code/pre  This occurs about six or seven lines in. Change this, and the description changes. The second entry is of the form:  precode&lt;key&gt;title&lt;/key&gt; &lt;string&gt;http://arxiv.org/pdf/1111.1763/arxiv.org&lt;/string&gt; /code/pre  and you can imagine what that does.  This procedure worked for me in Safari 5.1.1 in 10.7.2; I presume it should similarly work for you.  Two caveats: I have no idea what this might do to an iCloud-synced reading list. If the edit doesn’t get marked with the appropriate edit date, there could be some weird sync conflict brouhaha and Safari might explode. Also, I quit Safari while I messed around in codeBookmarks.plist/code. No idea what might happen if you leave it open; probably nothing. But y’know. You can’t be too careful.  br  This is an example full entry for a Reading List item in codeBookmarks.plist/code.  precode&lt;dict&gt;     &lt;key&gt;ReadingList&lt;/key&gt;     &lt;dict&gt;         &lt;key&gt;DateAdded&lt;/key&gt;         &lt;date&gt;2012-11-29T23:30:55Z&lt;/date&gt;         &lt;key&gt;DateLastFetched&lt;/key&gt;         &lt;date&gt;2012-11-29T23:30:55Z&lt;/date&gt;         &lt;key&gt;PreviewText&lt;/key&gt;         &lt;string&gt; **This is the short description** &lt;/string&gt;     &lt;/dict&gt;     &lt;key&gt;Sync&lt;/key&gt;     &lt;dict&gt;         &lt;key&gt;Key&lt;/key&gt;         &lt;string&gt;”C=1234567890"&lt;/string&gt;         &lt;key&gt;ServerID&lt;/key&gt;         &lt;string&gt;https://example@example.com+ABCDEFG12345&lt;/string&gt;     &lt;/dict&gt;     &lt;key&gt;URIDictionary&lt;/key&gt;     &lt;dict&gt;         &lt;key&gt;title&lt;/key&gt;         &lt;string&gt; **A web page that I’ve saved** &lt;/string&gt;     &lt;/dict&gt;     &lt;key&gt;URLString&lt;/key&gt;     &lt;string&gt;http://www.example.com/readinglist&lt;/string&gt;     &lt;key&gt;WebBookmarkType&lt;/key&gt;     &lt;string&gt;WebBookmarkTypeLeaf&lt;/string&gt;     &lt;key&gt;WebBookmarkUUID&lt;/key&gt;     &lt;string&gt; 1234567890 &lt;/string&gt; &lt;/dict&gt; /code/pre  <comment37305> Exactly what I was looking for!
