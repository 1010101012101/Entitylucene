How long does the DNS cache last in Mac OS X? <body> I know I can manually flush the DNS cache, e.g. with codesudo killall -HUP mDNSResponder/code.  But how long will the cache last if I don't do this? Does it reset on a reboot?  <answer93956> You can flush the local DNS cache with the following command:  codedscacheutil -flushcache/code  How long a DNS entry is cached typically depends on the TTL of that DNS record, which is configured by the DNS admin of the relevant hostname.  You can find out the (remaining) TTL of any given DNS record with the codedig/code command:  codedig apple.stackexchange.com/code  In the answer section you'll see the remaining TTL right next to the hostname (in this example 257 seconds).  precode;; ANSWER SECTION: apple.stackexchange.com. 257    IN  A   198.252.206.16 /code/pre
