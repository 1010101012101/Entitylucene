How can I automatically run an AppleScript on shutdown or logout? <body> I am more used to the Javascript style event handlers,  but is there a similar thing in applescript that listens to system events like logout or shutdown to execute code  or is there a way to have an applescript run on shutdown or logout?  I know about script timer but I would rather have something free.  All my script does is mute the volume for the next startup.  <answer74959> Launch agents can be used for this. See this tutorial on MacScipter. If you are not familiar with the process, Lingon offers a simple interface to create the file. Here is an example to launch a script named ZuluDeltaNiner.scpt on login.  precode&lt;?xml version="1.0" encoding="UTF-8"?&gt; &lt;!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"&gt; &lt;plist version="1.0"&gt; &lt;dict&gt;    &lt;key&gt;Label&lt;/key&gt;    &lt;string&gt;com.paul.applescript_Launchd&lt;/string&gt;    &lt;key&gt;ProgramArguments&lt;/key&gt;    &lt;array&gt;        &lt;string&gt;osascript&lt;/string&gt;        &lt;string&gt;/ZuluDeltaNiner.scpt&lt;/string&gt;    &lt;/array&gt;    &lt;key&gt;QueueDirectories&lt;/key&gt;    &lt;array/&gt;    &lt;key&gt;RunAtLoad&lt;/key&gt;    &lt;true/&gt;    &lt;key&gt;StartOnMount&lt;/key&gt;    &lt;false/&gt;    &lt;key&gt;WatchPaths&lt;/key&gt;    &lt;array/&gt; &lt;/dict&gt; &lt;/plist&gt; /code/pre  <answer74966> Logout hooks were deprecated in 10.4, but they still work in 10.8.  precodeecho $'#!/usr/bin/osascript\n\nset volume with output muted' &gt; ~/.logouthook chmod +x ~/.logouthook sudo defaults write com.apple.loginwindow LogoutHook ~/.logouthook /code/pre  codesudo defaults write/code modifies files in code/var/root/Library/Preferences//code. The program can't be inserted as an argument directly.  Trapping the exit signal only seems to work when logging out and not when shutting down.
