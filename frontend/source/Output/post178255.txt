Mac nl only takes lines 2048 chars long <body> I'm having an issue where the Mac version of nl will mark a new line after 2048 characters.  I tried it on linux, and it doesn't do this.  Any idea why it does on Mac, and how to fix it?  <answer178257> From codeman nl/code:   precodeBUGS      Input lines are limited to LINE_MAX (2048) bytes in length. /code/pre   To fix, you can either patch the source yourself (might need some work) or get the source for the Linux version (which is probably available on any GNU server).  <comment211175> A bare-bones replacement for `nl` is `awk '{print NR,$0}'`.
