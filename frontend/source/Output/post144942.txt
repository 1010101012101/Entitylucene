ln -fFs does not update symlink <body> I have two directories newdir and otherdir and would like to update a symlink, that initially pointed to otherdir to newdir.  When I run:  precode&gt; ln -s otherdir symlink  /code/pre  I see a symlink:  precode&gt; ls -al symlink symlink -&gt; otherdir /code/pre  When I run:  precode&gt; ln -sFf newdir symlink /code/pre  I would expect symlink - newdir, but I see the old link when I run:  precode&gt; ls -al symlink symlink -&gt; otherdir /code/pre  And I also noticed that a new symlink exists:   precode&gt; ls -al symlink/newdir symlink/newdir -&gt; newdir /code/pre  Is there a way to prevent ln from interpreting the destination path as parent directory ? Or is the only way to prevent this from happening to remove the link first ?  <answer144943> Use the -h option:   precode-h    If the target_file or target_dir is a symbolic link, do not follow it.        This is most useful with the -f option, to replace a symlink        which may point to a directory. /code/pre   precode$ ln -shf newdir symlink $ ls -al symlink symlink -&gt; newdir /code/pre
