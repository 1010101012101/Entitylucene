The launchd plist manual page mentions an inversion takes place if codeRunAtLoad/code is set :     strongSuccessfulExit /strong      If true, the job will be restarted as long as the program exits and with an exit status of zero. Others have struggled with this behaviour, see launchd ignores RunAtLoad=false.  <comment150454> It appears that my `KeepAlive` definition was implying `RunAtLoad`, and even overriding my attempt to set `RunAtLoad` to false. Not sure why this is the case, but taking out the `KeepAlive` block seems to have fixed the problem.
