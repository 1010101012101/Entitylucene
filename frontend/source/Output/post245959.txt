zless exits when reaching EOF <body> I run the command  precodezless some.log /code/pre  in OS X 10.11.5 El Capitan.  It closes when reaching EOF.  I checked aliases and I do not have LESS env variable set up but still it looks like codezless/code runs with code-Q/code or code-q/code parameter on.  codeless/code command runs just as expected not closing after reaching EOF.  Any help would be appreciated.  <answer246002> zless/zmore will use /usr/bin/more as the default PAGER unless PAGER is set. Add the following lines to your shell login/env file (bash, ksh, and zsh).  precodePAGER=/usr/bin/less export PAGER /code/pre  For tcsh:  precodesetenv PAGER /usr/bin/less /code/pre
