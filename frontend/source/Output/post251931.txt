The codedumpiso/code verb inspects and interprets ISO-9660 and Joliet structures. The format of the command is:  precodedrutil dumpiso &lt;device&gt; &lt;block&gt; [format] /code/pre  where the arguments are (see codeman drutil/code for details):  pre device   Disk node, e.g. /dev/disk1.  block    Block number to dump.  format   If format is not specified dumpiso will attempt to guess.           If present, this argument should be one of the following:          None, Boot, BootCat, PVD, SVD, VPD, VDST, MPath, LPath,          Dir, or HFSPlusVH. /pre  The codedrutil/code command allows a shortcut that avoids the need to attach, mount or detach an ISO image file. The command is:  precodedrutil dumpiso &lt;image&gt; &lt;block&gt; /code/pre  where code&lt;image&gt;/code is the name of an ISO image file. For example, codedrutil dumpiso image.iso 16/code displays block 16 from an image file named codeimage.iso/code and guesses the block's format.  <comment313381> Thank you for your answer, but I could not find a way to use drutil to make an output like isoinfo's. <comment313401> `drutil` and `isoinfo` are *similar* in that they both display ISO-9660 structures, but they are not *identical* so one can't expect the output to be the same. <comment313451> Thank you for your reply.
