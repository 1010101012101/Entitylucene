How to handle bundle identifiers with spaces? <body> Lets assume my current directory is code/Applications/code and it contains all my apps. Lets say I want to know the bundle identifier for Google Chrome:  precode$ mdls -raw -name kMDItemCFBundleIdentifier 'Google Chrome.app' com.google.Chrome /code/pre  Now let’s use that to locate the app:  precode$ mdfind kMDItemCFBundleIdentifier = com.google.Chrome /Applications/Google Chrome.app /code/pre  Now lets try the same for Tor Browser Bundle. First, get the bundle identifier:  precode$ mdls -raw -name kMDItemCFBundleIdentifier TorBrowser.app org.mozilla.tor browser /code/pre  Then, locate the app:  precode$ mdfind kMDItemCFBundleIdentifier = org.mozilla.tor browser /code/pre  Nothing. I likely need to quote it:  precode$ mdfind kMDItemCFBundleIdentifier = 'org.mozilla.tor browser' /code/pre  Nope, still nothing. Lets try something else:  precode$ mdfind kMDItemCFBundleIdentifier = 'org.mozilla.tor' $ mdfind kMDItemCFBundleIdentifier = "org.mozilla.tor browser" $ mdfind kMDItemCFBundleIdentifier = org.mozilla.tor\ browser $ mdfind kMDItemCFBundleIdentifier = "org.mozilla.tor\ browser" $ mdfind kMDItemCFBundleIdentifier = 'org.mozilla.tor%20browser' $ mdfind kMDItemCFBundleIdentifier = 'org.mozilla.tor+browser' $ mdfind kMDItemCFBundleIdentifier = 'org.mozilla.tor_browser' /code/pre  Nothing works. This is the only app I ever had this issue with, and the only one I ever saw with a space in its bundle identifier. How can we reference it?  <answer238381> A proper bundle identifier should strongnot/strong have a space in it. According to Apple's App Distribution Guide - About Bundle IDs, a     "...bundle ID string must be a uniform type identifier (UTI) that contains   only alphanumeric characters (A-Z,a-z,0-9), hyphen (-), and period   (.). The string should be in reverse-DNS format. For example, if your   organization’s domain is codeAcme.com/code and you create an app named codeHello/code,   you could assign the string codecom.Acme.Hello/code as your app’s bundle ID.   So, technically, codeorg.mozilla.tor browser/code is not a valid bundle identifier, so you'll get unexpected results when trying to work with it. It should likely be changed (by the developer) to something like codeorg.mozilla.tor.browser/code or codeorg.mozilla.tor-browser/code, etc.  <comment246311> I can't check it, because I do not have an app with space in bundle identifier, but how about escaping the space with `\\`` ?
