How are command line commands stored in .bash_history? <body> Command-line commands are collected in code.bash_history/code. Fine! But not all. Often I close a terminal and try a day later to find a command with kbdCtrl/kbdkbdR/kbd, but bash tells me "command not found". Why are some commands stored in history and some not? How can I prevent lost commands? I use OS X 10.10.5.  <answer283243> The history size is controlled by the variables HISTFILESIZE and HISTSIZE (see codeman bash/code). The default value is 500.   You must store these parameters in code~/.bash_profile/code  Example:  precodeexport HISTFILESIZE=5000 export HISTSIZE=5000 /code/pre  <comment357065> Read section [9.1 Bash History Facilities](https://www.gnu.org/software/bash/manual/html_node/Bash-History-Facilities.html#Bash-History-Facilities) in the [Bash Reference Manual](https://www.gnu.org/software/bash/manual/html_node/index.html).
