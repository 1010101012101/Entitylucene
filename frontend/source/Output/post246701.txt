What format does Xcode use to display 'data' types in plist files? <body> If I open a .plist file in TextEdit, all the properties with type "data" are displayed in Base64 encoding, for example code&lt;data&gt;SGFja2ludG9zaA==&lt;/data&gt;/code.  Xcode shows these values in some other format, like code&lt;54657374 56616c75 65&gt;/code.  What is this format called, and how can I convert it to ASCII?  <answer246709> The data is just a hex dump of the decoded base64 encoding. To convert to ASCII, look the hex values up in the ASCII table. For your example:  precode54657374 56616c75 65 /code/pre  is  precodeTestValue /code/pre  A convenient way to get the table is codeman ascii/code from a terminal prompt.
