How to run iPython Notebook as a service <body> What would be the best "standard" way to create a codelaunchd/code script to run codepython notebook/code?  I would like the service to start at login and an easy to start-stop-restart the server from a terminal.  <answer214905> In the end I created the following .plist file. It is autoloaded on startup and it will start a codejupiter/code notebook and keep it alive.  precode&lt;?xml version="1.0" encoding="UTF-8"?&gt;  &lt;!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"   "http://www.apple.com/DTDs/PropertyList-1.0.dtd"&gt;       &lt;plist version="1.0"&gt;       &lt;dict&gt;           &lt;key&gt;KeepAlive&lt;/key&gt;           &lt;true/&gt;           &lt;key&gt;Label&lt;/key&gt;           &lt;string&gt;cern.chernals.ipython&lt;/string&gt;           &lt;key&gt;ProgramArguments&lt;/key&gt;           &lt;array&gt;               &lt;string&gt;/usr/local/bin/jupyter-notebook&lt;/string&gt;           &lt;/array&gt;           &lt;key&gt;RunAtLoad&lt;/key&gt;           &lt;true/&gt;           &lt;key&gt;StandardErrorPath&lt;/key&gt;           &lt;string&gt;/Users/chernals/Library/LaunchAgents/jupyter-notebook.stderr&lt;/string&gt;           &lt;key&gt;StandardOutPath&lt;/key&gt;           &lt;string&gt;/Users/chernals/Library/LaunchAgents/jupyter-notebook.stdout&lt;/string&gt;               &lt;key&gt;KeepAlive&lt;/key&gt;               &lt;true/&gt;       &lt;/dict&gt;       &lt;/plist&gt; /code/pre  <answer266832> I'm a macOS newbie, and found the accepted answer very helpful. I modified it to not launch a browser at startup and also start in the directory ~/jupyter where I store most of my notebooks.   For future reference, the way to make this start is to put the xml data in a file in the directory ~/Library/LaunchAgents, e.g. ~/Library/LaunchAgents/org.jupyter.notebook.plist.  The plist I ended up with was the following.  precode&lt;?xml version="1.0" encoding="UTF-8"?&gt; &lt;!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"&gt; &lt;plist version="1.0"&gt; &lt;dict&gt;     &lt;key&gt;KeepAlive&lt;/key&gt;     &lt;true/&gt;     &lt;key&gt;Label&lt;/key&gt;     &lt;string&gt;org.jupyter.notebook&lt;/string&gt;     &lt;key&gt;ProgramArguments&lt;/key&gt;     &lt;array&gt;         &lt;string&gt;/usr/local/bin/jupyter-notebook&lt;/string&gt;         &lt;string&gt;--no-browser&lt;/string&gt;     &lt;/array&gt;     &lt;key&gt;RunAtLoad&lt;/key&gt;     &lt;true/&gt;     &lt;key&gt;StandardErrorPath&lt;/key&gt;     &lt;string&gt;/Users/rickard/Library/LaunchAgents/org.jupyter.notebook.stderr&lt;/string&gt;     &lt;key&gt;StandardOutPath&lt;/key&gt;     &lt;string&gt;/Users/rickard/Library/LaunchAgents/org.jupyter.notebook.stdout&lt;/string&gt;     &lt;key&gt;KeepAlive&lt;/key&gt;     &lt;true/&gt;     &lt;key&gt;WorkingDirectory&lt;/key&gt;     &lt;string&gt;/Users/rickard/jupyter&lt;/string&gt; &lt;/dict&gt; &lt;/plist&gt; /code/pre  I haven't figured out a way to not have to hard code the user's home directory in there -- the EnableGlobbing option only works in ProgramArgument, and there is no environment variable expansion, so for now it needs to be in there.  <comment366411> I've written a script that configures a launchagent like this automatically.
