This will indicate whether its a system / software issue rather than hardware - if the clean OS boots fine without a crash, it's not your hardware. Could save you time and expense.   If you don't have another OS to hand, try adding a new user with Admin rights but NO additional software. If this works the issue is limited to a an issue on the primary account. In the same vein, boot in safe mode - this will force diagnostics and turn all but essential software off - if the MacBook works OK in this mode, its another pointer to an OS / account / software issue.  It's worth noting that older Macs don't behave well with a significant OS update through migration assistant e.g. 10.7 to Sierra. You can end up with a mix of old and new components which often cause beach ball stalls and crashing. Any major OS update should be on a clean drive i.e. wipe / fresh install of the OS and software - the only way to have a fast, optimised system. Since you have bought the Mac used, don't assume the OS is a clean install.  In 20 years of maintaining Macs, 9 times out of 10, problems are (in this order): • OS badly installed, updated or maintained OR unsuitable for the hardware (even if the latest OS will install, doesn't mean you should upgrade - performance most often drops, especially on Macbooks that are already de-tuned for heat, noise and battery life).  • Ram - not enough, modern systems can use 4 to 6GB before you begin work • fonts - too many, badly installed within the system folder, conflicting • existing software incompatible with new OS • startup items, including Akamai, Microsoft and Adobe updaters that run constant process in the background - turn them off. Use Utlities / Activity Monitor to see what is running in the background. • web browsers running Flash especially with Safari on older MacBooks / OS's - will hog Ram, speed up the fans and reduce the MacBook to a crawl. • language kits and printer support - delete all that you don't use - will save you 10+GB of HDD and potentially improve drive access speed.  I'm not saying it's not a graphics card issue or other hardware fault, although its worth discounting the common issues first, before you run to the expense of an 'expert' repair (or return).  <comment320060> Did you wipe and reinstall the OS or is this the OS that was loaded when you purchased the Mac? <comment320061> The GPU panic seems worth ruining diagnostics whether this model has a repair extension program or not.j Worst case, mail in repairs might be $300 and then you could contact the seller for a refund or price adjustment if you need to pay for a repair right after you purchased the Mac. <comment327783> For those wondering, it was indeed a hardware issue with a defective model.
