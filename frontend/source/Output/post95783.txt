codeport list/code shows emall/em packages that are known to MacPorts. To get a list of packages that are actually eminstalled/em, use codeport installed/code. Slower but with a few more details: codeport list installed/code.  As for your actual problem: Maybe the reason why uninstalling "texlive" produces no output is because the package is not installed in the first place. Try to determine a concrete binary that is interfering with MacTex and that you want to get rid of. Then determine the MacPorts package to uninstall with this command: codeport provides /path/to/file/code.  <answer133129> I was trying to solve the same problem because I have a working MacTeX install, but then macports installed texlive as a dependency (for auctex).
