Cannot mount UDF formatted hard disk in OSX <body> I have formated a 2TB external disk with UDF filesystem in Ubuntu. The way I formatted it was:  precode   sudo dd if=/dev/zero of=/dev/sdc bs=2048 count=1    sudo mkudffs --media-type=hd --vid=myDisk --blocksize=2048 /dev/sdc /code/pre  Now I plug the disk in OSX and nothing is automount. The disk is recognized, but nothing else, it's not mounted. I am in OSX Snow Leopard 10.6.8.  codediskutil list/code shows:  precode /dev/disk2  #:                       TYPE NAME                    SIZE       IDENTIFIER  0:                                                   *2.0 TB     disk2 /code/pre  So first problem is that it doesn't recognize the TYPE NAME is defined in format option (--vid=myDisk). Anyway....  I tried to mount the disk manually:  precode   sudo mkdir /Volumes/myDisk    sudo mount -v -t udf /dev/disk2 /Volumes/myDisk/ /code/pre  And I get:  precode mount_udf: /dev/disk2 on /Volumes/myDisk: Device not configured /code/pre  If I try:  precode diskutil mount  -mountPoint /Volumes/myDisk /dev/disk2 /code/pre  I get  precode Volume on disk2 failed to mount; if it has a partitioning scheme, use "diskutil mountDisk" /code/pre  If I try:  precode sudo mount_udf -b 2048  /dev/disk2 /Volumes/myDisk/ /code/pre  I get  precode   mount_udf: /dev/disk2 on /Volumes/myDisk/: Device not configured /code/pre  Any ideas?
