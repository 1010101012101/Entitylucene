How to run a command at startup <body> I want to automatically start my MongoDB Server at startup. The command for it is  precodemongod --dbpath="/Users/prateek/Desktop/Mongo Database" /code/pre  How do I configure this in Mac OSX El Capitan?  Note - I have installed MongoDB using Homebrew  <answer226000> Create a file named org.mongod.user.plist in ~/Library/LaunchAgents with the content:  precode&lt;?xml version="1.0" encoding="UTF-8"?&gt; &lt;!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"&gt; &lt;plist version="1.0"&gt; &lt;dict&gt;     &lt;key&gt;Label&lt;/key&gt;     &lt;string&gt;org.mongod.user&lt;/string&gt;     &lt;key&gt;ProgramArguments&lt;/key&gt;     &lt;array&gt;         &lt;string&gt;mongod&lt;/string&gt;         &lt;string&gt;--dbpath=/Users/prateek/Desktop/Mongo Database&lt;/string&gt;     &lt;/array&gt;     &lt;key&gt;RunAtLoad&lt;/key&gt;     &lt;true/&gt; &lt;/dict&gt; &lt;/plist&gt; /code/pre  This requires that emmongod/em is in your path.  If mongod is not in your path, use the whole path in the first string of the ProgramArguments array (e.g. em/usr/local/Cellar/mongodb/3.2.1/bin/mongod/em)  If you need stdout or stderr (which can be helpful if something fails) add the following in the dict section of the plist:  precode    &lt;key&gt;StandardErrorPath&lt;/key&gt;     &lt;string&gt;/tmp/org.mongod.user.stderr&lt;/string&gt;     &lt;key&gt;StandardOutPath&lt;/key&gt;     &lt;string&gt;/tmp/org.mongod.user.stdout&lt;/string&gt; /code/pre  Load and start mongod with:  precodelaunchctl load ~/Library/LaunchAgents/org.mongod.user.plist launchctl start org.mongod.user /code/pre  hr  To connect to your MongoDB database enter:  precodemongo localhost /code/pre  Again this requires that emmongo/em is in your path. Otherwise use the complete path (e.g. em/usr/local/Cellar/mongodb/3.2.1/bin/mongo/em).  <comment274354> See the SO question http://stackoverflow.com/questions/5596521/what-is-the-correct-way-to-start-a-mongod-service-on-linux-os-x <comment274829> This is the way to do it.
