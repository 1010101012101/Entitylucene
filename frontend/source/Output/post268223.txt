Creating encrypted disk image passphrase in Keychain, and strange GUID account <body> When a user persists an encrypted disk (or disk image) passphrase in the Keychain (e.g via Disk Utility as shown here, https://apple.stackexchange.com/a/229309/5392 or when setting up an encrypted Time Machine backup, possibly on a remote device), an entry gets created in either the login (via Disk Utility) or System (for Time Machine) keychains.  img src="https://i.stack.imgur.com/812Zm.png" alt="Keychain screenshot"  Examining such an entry with codesecurity/code provides the following data:  precode$ security find-generic-password -D "disk image password" -l "A65D7474-F452-51CB-9BCC-3D594C918978.sparsebundle" keychain: "/Library/Keychains/System.keychain" version: 256 class: "genp" attributes:     0x00000007 &lt;blob&gt;="A65D7474-F452-51CB-9BCC-3D594C918978.sparsebundle"     0x00000008 &lt;blob&gt;=&lt;NULL&gt;     "acct"&lt;blob&gt;="17F24503-E0E2-45B9-86EA-C79D857BA2AC"     "cdat"&lt;timedate&gt;=0x32303137303131303132333130335A00  "20170110123103Z\000"     "crtr"&lt;uint32&gt;=&lt;NULL&gt;     "cusi"&lt;sint32&gt;=&lt;NULL&gt;     "desc"&lt;blob&gt;="disk image password"     "gena"&lt;blob&gt;=&lt;NULL&gt;     "icmt"&lt;blob&gt;=&lt;NULL&gt;     "invi"&lt;sint32&gt;=&lt;NULL&gt;     "mdat"&lt;timedate&gt;=0x32303137303131303132333130335A00  "20170110123103Z\000"     "nega"&lt;sint32&gt;=&lt;NULL&gt;     "prot"&lt;blob&gt;=&lt;NULL&gt;     "scrp"&lt;sint32&gt;=&lt;NULL&gt;     "svce"&lt;blob&gt;="A65D7474-F452-51CB-9BCC-3D594C918978.sparsebundle"     "type"&lt;uint32&gt;=&lt;NULL&gt; /code/pre  The account's GUID matches none of the users that can be found via:  precodedscl /Search -list /Users GeneratedUID /code/pre  The ultimate goal is to non-interactively create the necessary entry in the Keychain for Time Machine to pick up the password and attach an encrypted, size-limited sparsebundle disk image on a Time Capsule, for which this is the missing piece.  Unfortunately codediskimages-helper/code is not allowed to mount it (observed via codeConsole.app/code) when using the following command to create the entry:  precodesudo security add-generic-password -a "$username" -w "${passphrase}" -D "disk image password" -s "${sparsebundle}" -T /System/Library/PrivateFrameworks/DiskImages.framework/Versions/A/Resources/diskimages-helper /Library/Keychains/System.keychain /code/pre  BTW this is on Sierra.
