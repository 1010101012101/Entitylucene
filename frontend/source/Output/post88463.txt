If the system hostname has been set, changing the computer name or local hostname from System Preferences won't change the prompt.  <comment103544> THanks! changing the computer name solved the problem! <comment151754> +1; I conclude that there's no good reason to ever set `HostName` (despite what `man hostname` says!), given that its value is then out of sync with the `LocalHostName` value set via System Preferences. Conversely, `LocalHostName` _is_ also recognized by `hostname`. To put it another way: if you want to change the hostname from a shell, use BOTH `sudo scutil --set LocalHostName newName` and `sudo scutil --set ComputerName newName`, where the `ComputerName` value can be a friendlier version (spaces, punctuation) of the `LocalHostName` value (alphanumerics and dashes only). <comment151755> To get rid of a `HostName` entry, set its value to an empty string (this effectively removes the key from the `.plist` file): `sudo scutil --set HostName ''`.
