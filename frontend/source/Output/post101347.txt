Verify Time Machine Encryption from the Command-Line? <body> Is it possible to verify whether a given Time Machine volume (for example, returned by codetmutil machinedirectory/code) has encryption turned on?  This is for regulatory compliance, so it needs to be fully scriptable.  <answer101597> If you know the UUID before you mount the disk you can use:  precodediskutil coreStorage list XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX /code/pre  This will give the following output.  precodeLogical Volume Family XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX ---------------------------------------------------------- Encryption Status:       Unlocked Encryption Type:         AES-XTS Conversion Status:       Complete Conversion Direction:    -none- Has Encrypted Extents:   Yes Fully Secure:            Yes Passphrase Required:     Yes /code/pre  If you know the mountpoint of the disk, you can get the UUID by the following command.  precodediskutil info /MOUNTPOINT | grep UUID | awk '{print $3}' /code/pre  And if you do not know the mountpoint you can always run  precodediskutil coreStorage list /code/pre  A MacMini with two encrypted drives (Macintosh SSD &amp;
